## <서비스 플로우(Service Flow)>
### 전체적인 서비스 흐름 : 사용자 경험 중시
![[서비스 플로우.png]]
1. 상시 대기 및 시각 정보 제공
    - 기기 화면에 현재 시간, 날씨, 오늘의 일정이 상시 표시
    - 사용자의 입력이 없을 때는 대기(Idle) 상태를 유지
    - 1시간 동안 움직임이 감지되지 않으면 화면이 어두워지고 캐틱터가 절전 모드로 전환
2. AI 호출
    - 사용자가 “퍼비야”라고 음성 호출
    - 기기가 이를 감지하면, 화면 속 캐릭터가 사용자 방향으로 집중하며 귀를 기울이는 동작(Listening)으로 상태 변화
3. 명령 입력 및 AI 처리
    - 사용자가 음성으로 원하는 기능을 명령(예 : 일정 제어, 음악 재생, 타이머 설정, 하루 일과 회고 등)
    - AI가 사용자의 명령을 분석하고 처리하는 동안, 화면 속 캐릭터는 생각하는 표정과 함께 생각 구름이나 로딩링(Thinking)을 띄움
4. 결과 피드백
    - 처리가 완료되면 화면 UI가 업데이트되어 시각적인 결과(예 : 추가된 일정 표시)를 보여주고, 스피커를 통해 AI의 음성 답변이 출력
    - 만약 AI가 명령을 이해하지 못했다면, 당황한 표정과 머리 위 물음표(Error)를 띄우며 오류 메시지를 표시
5. 모바일 앱 연동
    - 사용자는 모바일 앱을 통해 기기와 연결하여 배경이나 AI 설정, AI의 이름 등을 실시간으로 제어하고 관리 가능


## <논리적 아키텍처 다이어그램(Logical Architecture Diagram)>
![[논리적 아키텍처.png]]
### 로컬 디바이스 영역
- 물리적 입출력 장치
    - 디스플레이 : 화면 출력
    - 마이크 : 사용자의 음성 입력
    - 스피커 : AI 목소리 출력
- 웹 뷰어(Web Viewer) : 라즈베리파이 내의 웹 브라우저를 통해 메인 서버에 배포된 프론트엔드 웹사이트에 접속하여 화면(시간, 날씨, 일정, 애니메이션 등)을 상시 띄워두는 역할
- 음성 전처리 모듈
    - 호출어(Wake Word) 감지 모듈 : 마이크로 들어오는 소리를 분석하여 AI 이름을 감지
    - 로컬 음성 인식(STT) 모듈 : 호출어 이후의 음성을 텍스트로 변환(지연 시간 감소 목적)
- 로컬 제어 서버(Main Controller) : 변환된 음성 텍스트를 메인 서버로 전송하고, 메인 서버로부터 받은 결과(오디오 출력 명령 등)를 하드웨어에 전달하는 역할

### 메인 서버 영역(중앙 제어, 상태 관리 및 웹 호스팅)

- API 통합 수신 모듈 : 로컬 디바이스(라즈베리파이), 웹 프론트엔드, 모바일 앱에서 들어오는 모든 요청을 처리하는 중앙 접점
- 상태 관리 및 데이터베이스 연동 모듈 : 사용자의 일정, AI의 이름, 배경 설정 등의 데이터를 데이터베이스에 저장하고 관리
- AI 프롬프트 생성 및 제어 로직 : 전달받은 텍스트에 로컬 DB의 정보를 결합하여 외부 AI 엔진으로 전달
- 웹 프론트엔드 호스팅 : 디스플레이에 띄워질 UI 웹 애플리케이션을 배포하고 제공하는 역할

### 외부 클라우드 서비스 영역

- 대화형 AI 엔진(LLM) : 텍스트의 의도를 파악하고 자연스러운 답변 생성
- 음성 합성 엔진(TTS) : 답변 텍스트를 오디오 파일로 변환

### 모바일 클라이언트 영역(사용자 설정 관리)

- 모바일 관리 앱 : 사용자가 스마트폰을 통해 메인 서버 API와 통신하여 기기의 배경, AI 이름, 시스템 설정 등의 데이터를 수정하고 관리하는 클라이언트 애플리케이션

>[!quote]+ AI 음성 명령 처리 
>1. 사용자 발화 → 마이크: 사용자가 "퍼비야"라고 말하여 오디오 데이터가 입력
>2. 로컬 제어 서버 내부 처리 : Wake Word 모듈이 오디오를 분석해 호출어 감지
>3. **로컬 제어 서버 → API 통합 수신 모듈(메인 서버)** : 호출어가 감지되면 라즈베리파이가 메인 서버에게 “듣기 시작함”이라는 상태 신호(API 요청)를 전송
>4. **메인 서버 → 웹 뷰어** : 메인 서버는 접속해 있는 웹 뷰어 UI 상태를 ‘듣기’ 애니메이션으로 업데이트하도록 제어
>5. **사용자 발화 → 로컬 제어 서버 내부 처리** : 사용자가 “오늘 일정 알려줘”라고 이어서 말하면, 라즈베리파이 내부의 STT 모듈이 입력된 음성을 텍스트로 변환
>6. 로컬 제어 서버 → API 통합 수신 모듈(메인 서버) : 변환이 완료된 텍스트 데이터(”오늘 일정 알려줘”)를 메인 서버로 전송
>7. **API 통합 수신 모듈(메인 서버) → 대화형 AI 엔진(LLM)** : 수신한 텍스트 데이터의 의도(intent)를 분석하고, 데이터베이스 조회를 위한 파라미터 또는 쿼리문을 생성해 달라고 요청
>> - 인텐트 분석(인텐트 추출) : 자연어에서 사용자의 목적을 추출하는 과정
>8. 대화형 AI 엔진(LLM) → 메인 서버 : 분석된 의도와 쿼리 조건이 메인 서버로 반환
>9. 상태 관리 및 데이터베이스 모듈(메인 서버) 내부 처리 : LLM으로부터 반환받은 쿼리 조건(의도)을 사용하여 데이터베이스(DB)에 쿼리문을 실행하고, 해당하는 실제 일정 데이터를 조회하여 가져옴
>10. 상태 관리 및 데이터베이스 연동 모듈(메인 서버) → AI 프롬프트 생성 로직(메인 서버) : 조회한 DB 데이터와 사용자의 질문 텍스트를 결합하여 최종 텍스트 프롬프트 조립
>11. AI 프롬프트 생성 로직(메인 서버) → 대화형 AI 엔진(LLM) : 완성된 텍스트 프롬프트를 외부 클라우드로 보내어 의도 분석 및 자연스러운 답변 생성 요청
>12. 대화형 AI 엔진 → 메인 서버 : 완성된 답변 텍스트가 메인 서버로 반환
>13. 메인 서버 → 음성 합성 엔진(TTS) : 반환된 텍스트를 외부 클라우드로 보내 오디오 파일 변환을 요청하고, 변환된 오디오 데이터를 다시 메인 서버가 받음
>14. 메인 서버 → 로컬 제어 서버 & 웹 뷰어
>> - **웹 뷰어** : 화면에 생성된 일정 텍스트를 띄우고 상태를 다시 대기(Idle) 모드로 변경
>> - **로컬 제어 서버** : 전달받은 오디오 데이터를 스피커를 통해 음성 출력
